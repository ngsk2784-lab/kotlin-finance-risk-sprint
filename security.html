01 / ACCESS

접근 범위를 먼저 고정합니다.

저장소 · 커밋 · 서비스 경로
READ

검사 단계는 읽기 전용

DEFAULT

대상 저장소·서비스로 제한된 임시 읽기 전용 권한을 사용하고, 고정 커밋 SHA와 포함·제외 경로를 SOW에 기록합니다.

NOT ACCEPTED

전체 조직 관리자, 공유 계정, 장기 개인 토큰, 프로덕션 셸·DB·클라우드 콘솔 권한은 받지 않습니다.

WRITE

PR은 별도 승인된 임시 브랜치만

WHEN ALLOWED

리미디에이션 경로 A를 선택하고 고객이 PR을 원할 때만 임시 작업 브랜치의 최소 쓰기 권한을 별도로 승인받습니다.

BOUNDARY

기본 브랜치 병합과 운영 배포는 고객이 수행합니다. 패치 파일·플레이북 경로에는 쓰기 권한이 필요하지 않습니다.

02 / DATA HYGIENE

받지 않는 데이터를 명시합니다.

실데이터 없이 재현
CREDENTIALS×운영 비밀번호·키·토큰·개인키
PERSONAL DATA×주민번호·계좌·카드·연락처
PRODUCTION×실거래 로그·DB 덤프·고객 데이터
발견 시 복사하지 않고 중단

실제 시크릿이나 개인·금융 데이터가 범위에서 발견되면 값을 보고서에 복사하거나 인용하지 않습니다. 작업을 멈추고 위치만 최소한으로 알리며, 회전·폐기와 사고 대응 판단은 고객이 수행합니다.

03 / EXTERNAL PROCESSING

외부 AI·모델 입력은 기본 허용이 아닙니다.

승인 전 전송 금지
확인 항목SOW 기록승인 전 상태
제공자·제품·처리 목적실제 사용 대상과 전송 범위금지
보존·학습 사용 여부정책 URL·버전·확인일금지
하위처리자·처리 위치확인 근거와 예외금지
고객 서면 승인승인자·승인일·허용 범위필수
공개 입력·링크는 사용하지 않음

고객 승인 여부와 무관하게 원본 코드·산출물을 공개 AI 입력, 공개 저장소 또는 공개 파일 링크에 올리지 않습니다.

05 / INCIDENTS

의심되면 먼저 멈춥니다.

사실·시각·영향 범위 기록
복사 방지 → 작업 중지 → 고객 통지

무단 접근, 잘못된 수신자, 시크릿 노출 또는 범위 밖 데이터가 의심되면 추가 복사를 막고 작업을 중지합니다. 통지 기한과 방식은 고객 정책과 적용 법률을 SOW에서 확인합니다. 기술적으로 검증하지 못한 “완전 복구 불가” 또는 인증 삭제를 주장하지 않습니다.

공개 운영 원칙 · 법률 자문 아님

이 페이지는 보안·데이터 처리 기본안을 설명하며 법률 자문, 인증 또는 특정 관할 준수 보장이 아닙니다. 실제 거래 전 당사자 지위, 고객 정책, 적용 법률과 최종 SOW를 확인합니다.
